Multi-layer agentic pipelines — in which an LLM agent retrieves context via a standardised tool protocol, synthesises a response, and renders it through a structured user interface — are increasingly deployed in practice, yet remain poorly understood from an evaluation standpoint. Existing work evaluates such systems at the pipeline level, scoring only the final output against a reference; no framework characterises where information fidelity is gained or lost between architectural layers. We propose layer-wise fidelity scoring: BERTScore Precision, Recall, and F1 computed independently at R0 (raw MCP tool output), R1 (A2A agent synthesis), and R2 (A2UI structured payload), each against the same ground-truth reference answer. We instantiate this protocol on a software documentation Q&A task using the httpx Python library, building a minimal end-to-end prototype comprising an MCP retrieval server, a LangGraph A2A orchestrator, and a Streamlit-based rendering layer that simulates A2UI structured output. Experiments across 30 test queries show that A2A synthesis significantly improves BERTScore F1 (Δ1 = +0.038, p < 0.001) by raising Precision while leaving Recall essentially unchanged, confirming that synthesis improves content selectivity without sacrificing coverage. The A2UI formatting step is near-lossless on F1 with only minor Recall compression. These layer-level dynamics are invisible to aggregate end-to-end scoring and only emerge through layer-wise analysis. Our protocol is task-agnostic and reusable; we release the annotated test set and scoring harness to support future benchmarking of UI-terminal agentic systems.
